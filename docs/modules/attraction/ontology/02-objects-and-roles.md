# 02. Объекты, Роли И Источники

## Главные Объекты

| Объект | Определение |
| --- | --- |
| Потенциальный участник | Человек, которого рассматривают для одного из бизнес-клубов. Это главный бизнес-субъект аналитики. |
| Сделка привлечения | CRM-оболочка работы с потенциальным участником. Стадия сделки показывает процессное положение, но не всегда доказывает бизнес-факт. |
| Поставка | Передача потенциального участника из источника в `База входящая`. Для Лидген УС поставка имеет итоговое качество. |
| Итоговое качество | Качество, в котором источник передает потенциального участника в привлечение: например, `Готов ко встрече` или `Готов к мероприятию`. |
| Встреча-знакомство | Целевой этап для организации и проведения встречи знакомства. Встреча обычно приоритетно проводится в Zoom для Привлечения штрих. |
| Конверсионное событие | Формат, который должен изменить состояние участника: активировать, продемонстрировать ценность, проблематизировать запрос или провести к адмиссии. |
| Задача | Операционный следующий шаг по сделке: канал, цель, дата и время. Открытая сделка без следующей задачи считается неконтролируемой. |
| Возврат | Ранняя маршрутизация из-за неквалификации или нарушения условий поставки до полноценной работы КИ. |
| Корзина | Текущий проигрыш после работы КИ: клиенту сейчас не ценно, он отказался или не дошел до членства. |
| Прогрев | Отложенный повторный маршрут для потенциально релевантного человека, к которому есть смысл вернуться позже. |
| Отклонено потребителем | Отказ принимающего клуба принять участника/карточку на этапе `На передаче`, в том числе из-за неполных данных. |

## Роли

| Роль | Ответственность |
| --- | --- |
| Комьюнити-интегратор / КИ | Берет сделку в работу, подтверждает качество, ведет коммуникацию, организует встречу, проводит через события и фиксирует следующий шаг. |
| Менеджер старого привлечения | Работает в старом контуре, где ценность в большей степени продается через личную коммуникацию и презентацию. |
| Лидген УС | Отдельный модуль-поставщик, который квалифицирует потенциальных участников и передает их в согласованных итоговых качествах. |
| Владелец партнерского мероприятия | Организует лидогенерационное событие, где КИ может установить контакт с квалифицированными участниками. |
| Владелец формата | Отвечает за качество конверсионного события и ожидаемый эффект для потенциального участника. |
| Клуб / принимающая команда | Принимает участника после успешного привлечения. |
| Владелец онтологии | Удерживает термины, границы, stage map, причины, правила отчетности и синхронизацию с Битрикс. |

## Источники Входящих Сделок

| Источник | Как попадает в работу | Особенности |
| --- | --- | --- |
| Лидген УС | Поставка в `База входящая` с итоговым качеством. | Для входящих поставок запускается автораспределение сделок по очереди между контурами и внутри команды. |
| Внутренняя база | КИ может достать сделки в работу при малом количестве активных сделок. | Итогового качества пока нет. Автораспределение не запускается. |
| Рекомендация от участника | Рекомендация приходит вне стандартной поставки Лидген УС. | Нужна отдельная фиксация источника и качества. |
| Рекомендация от сотрудника | Сотрудник рекомендует потенциального участника. | Нужна отдельная фиксация источника и ответственного. |
| Самостоятельно созданная сделка | КИ сам создает сделку. | Автораспределение не запускается. |

## Первичные События Сделки

Эти события лучше описывать обычным языком, а не техническими кодами:

| Событие | Бизнес-смысл |
| --- | --- |
| Поставка пришла | Сделка появилась в `База входящая` из источника. |
| Ответственный назначен | У сделки появился КИ, который отвечает за следующий шаг. |
| Дедлайн взятия в работу установлен | Для релевантных поставок включился 24-часовой SLA. |
| Сделка взята в работу вручную | КИ перевел сделку из `База входящая` на этап `Звонок-знакомство`. |
| Сделка автокуплена | Из-за нарушения SLA сделка автоматически ушла на `Встреча-знакомство`, и покупка у Лидген УС считается принятой. |
| Первый звонок сделан | КИ совершил первую попытку связи в регламентный срок. |
| Первый звонок просрочен | Первая попытка связи не сделана в SLA. |

Эти события могут стать техническими facts в аналитике, но в онтологии они
сначала фиксируются как бизнес-факты.
