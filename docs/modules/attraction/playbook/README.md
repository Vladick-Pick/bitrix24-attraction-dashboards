# Плейбук Комьюнити-Интегратора

Операционный справочник для ввода комьюнити-менеджеров (КМ) в роль
комьюнити-интеграторов (КИ) модуля «Привлечение».

- Файл: [`playbook-ki.html`](./playbook-ki.html) — самодостаточный HTML
  (вкладки, карточки, фильтруемый каталог), открывается в браузере без сервера.
- Стиль: тот же визуальный язык, что и
  `Конверсионные-события-свод.html` в корне репозитория.

## Вкладки

| Вкладка | Что покрывает |
| --- | --- |
| Старт | Роль КИ, что меняется при переходе из КМ, карта процесса, легенда. |
| 1 · Сделки по этапам | Что делать на каждом этапе: цель, действия КИ, SLA, хороший выход, следующий шаг, поля, дефект; маршруты проигрыша. |
| 2 · Поля по этапам | Блоки карточки, чек-листы «что обязательно перед переводом на этап», дисциплина задач и фиксации. |
| 3 · Звонки | Коммуникационная рамка: скрипты входящих лидов и работы с базой, возражения, сообщения, фиксация состояния. |
| 4 · Встреча | Устройство встречи-знакомства: формат, тайминг, скрипт по блокам, рефлексия. |
| 5 · События | Паспорта состояний с триггерами, каталог 16 форматов, статусы участия, как приглашать. |
| Открытые вопросы | Всё неустоявшееся, вынесенное из ядра. |

## Принцип ядра и приложения

В рабочих инструкциях (вкладки 1–5) — **только утверждённое** в онтологии и
регламентах (бейдж `утверждено`). Всё неустоявшееся (точные названия стадий,
механика «Прогрева», расчётная конверсия, маршрутизация, точные сроки протухания,
коды полей) помечено бейджем `на уточнение` и собрано на вкладке «Открытые
вопросы». По таким пунктам КИ **не действует как по правилу** до закрепления
владельцем онтологии.

## Как обновлять

1. Менять самый узкий блок: при изменении правила — соответствующую вкладку;
   при изменении канонического термина — синхронизировать с онтологией.
2. Когда пункт из «Открытых вопросов» закрепляется — переносить его в ядро
   (нужную вкладку) и снимать бейдж `на уточнение`, параллельно обновляя
   онтологию (`02-state-layer`, registry, changelog).
3. Каталог форматов редактируется в массиве `DATA` внутри `playbook-ki.html`
   (источник — таблица «Конверсионные события»).
