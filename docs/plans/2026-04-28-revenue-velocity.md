# План переработки вкладки "Денежная скорость"

**Цель:** переделать PR #12 из когортного отчёта в денежную приборную панель состояния системы. Период по умолчанию трактуется как окно наблюдения, а не как дата создания сделки.

**Ключевые решения:**
- View по умолчанию: `systemState`.
- Дополнительные view: `operationalPeriod` и `createdCohort`.
- Тяжёлый отчёт загружается лениво при открытии вкладки и при смене view/разреза.
- Основные UI-названия метрик на русском языке.

## Backend

1. Факт продаж периода считать через `wonAt`:
   - первый переход в выигранную стадию по `stageHistory` внутри периода;
   - либо дата закрытия внутри периода, если текущая стадия сделки выигранная.
2. Активную воронку считать только по бизнес-активным стадиям:
   - База входящая;
   - Звонок-знакомство;
   - Встреча-знакомство;
   - Активация;
   - Проблематизация;
   - Демонстрация;
   - Адмиссия;
   - Контрактация;
   - На передаче.
3. Исключить из активной воронки терминальные и возвратные стадии:
   - Корзина;
   - Возврат в лидген;
   - Неквал;
   - Проиграно;
   - Отказ;
   - Передано в клуб / won / handoff.
4. Не показывать ожидаемые деньги воронки и денежную скорость, если нет достаточной исторической базы по стадиям. В этом случае возвращать `null` и предупреждение.
5. Узкое место выбирать только среди активных бизнес-стадий. Корзина и Возврат в лидген могут быть сигналами отвала/качества входа, но не bottleneck.
6. Для lifetime/cohort действий закрывать окно действий на `min(dateClosed, asOf)`, чтобы пост-продажные действия не портили деньги на действие.

## Frontend

1. Переключатель view: `Состояние системы`, `Оперативно`, `Когорты`.
2. Убрать статические placeholder KPI из scene-конфига для вкладки.
3. Верхние KPI для `systemState`:
   - Факт денег периода;
   - Активная воронка;
   - Денежная скорость;
   - Изменение скорости;
   - Системный прирост;
   - Действия, баллы;
   - ₽ системного прироста / балл;
   - Оценка будущих денег от действий.
4. Основная таблица по умолчанию должна иметь не больше 8 колонок.
5. Продвинутые метрики показывать в раскрытии строки, а не в основной таблице.
6. Формульные подсказки писать простым русским языком.

## Tests

1. Продажа периода учитывает переход в выигранную стадию внутри периода, даже если сделка создана раньше.
2. Fixture продажи Потаповой попадает в продажи периода.
3. Активная воронка исключает Корзину, Возврат в лидген, won/handoff, lost/refusal.
4. Количество активных сделок и WIP использует clean active stages.
5. Ожидаемые деньги воронки не получают молчаливый fallback 50% без калибровки.
6. Узкое место не возвращает Корзину или Возврат в лидген.
7. Default UI рендерит не больше 8 основных колонок.
8. Продвинутые метрики доступны только в раскрытии строки.
9. Отчёт не грузится до открытия вкладки.
