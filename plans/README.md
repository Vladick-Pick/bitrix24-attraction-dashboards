## Dependency Notes

- 001 must land first because it records the decision every shared/platform
  follow-up must obey: `attraction` and `leadgen` are separate products, not two
  skins of one report.
- 002 should land before 003 because route registrars should depend on narrow
  storage roles, not on the full `SqliteRepository` adapter.
- 003 is the first code split that makes module disablement real at the HTTP
  interface without physically forking the repository yet.
- 004 should land after 002 and 003 so future modules and AI-agent consumers use
  explicit storage/route seams instead of direct repository or report internals.
- 005 should land after 004 because the manifest must exist before fork-owned
  modules can bring their own manifest adapter, live route availability, source
  data, and integration rules. It keeps the project source-code-fork friendly
  without introducing a package/plugin runtime yet.
- 006 follows 002 because the repository role file now exists, but the reporting
  service still depends on the full `SqliteRepository`.
- 007 is independent and can run before or after 006. It touches only the web
  runtime and leadgen tests.
- 008 follows 003 because route registrars exist; the remaining issue is handler
  ownership inside `createApp`.
- 009 is independent. It is safest after no active source-quality report work is
  in flight, but it does not depend on 006-008.

Recommended execution order for the 2026-06-15 follow-up set:

1. 006 - narrows the server reporting contract before more route/service work.

3. 009 - optimizes a focused report builder with strong existing tests.

## Findings Considered And Rejected

- Full repository fork now: rejected for this sequence because it would copy the
  current `createApp`, `SqliteRepository`, and `ProtoApp` coupling into two
  repositories.
- Full web runtime refactor now: deferred. `docs/architecture/web-runtime.md`
  requires `ProtoApp` to remain the one supported runtime interface; internal
  web adapters can come after server/storage seams.
- Full contracts package split now: deferred. It is useful, but less blocking
  than storage roles and route ownership for safe repository extraction.
- Package/plugin runtime now: deferred. Open-source forkability should first use
  a source-level extension seam; dynamic package loading can come later if real
  fork maintainers need it.
- Call-analysis transcript visibility as an urgent bug: rejected as a bug because
  the current full-transcript visibility is documented as an explicit MVP risk,
  not accidental behavior. Track as a separate privacy-hardening/product
  governance task when moving beyond MVP.
- Dependency hygiene as a top-four fix: deferred. `pnpm audit --prod` currently
  reports a moderate transitive `qs` advisory through `express`, with no
  high/critical production advisory observed during the 2026-06-15 review.
